Telemetry Trainee Program — Rocketry

Torne-se um colaborador ativo do time de telemetria

Objetivos de Aprendizado

  • Configurar o ambiente de desenvolvimento completo
  • Dominar o fluxo básico do Git (clone, branch, commit, push)

Setup do Ambiente

Ferramentas Essenciais

  • VSCode — editor principal com extensões:
    • Python, Pylance, Jupyter
    • Arduino, PlatformIO IDE
    • GitLens, GitHub Pull Requests
  • Python 3.10+ — análise de dados
  • Arduino IDE — firmware embarcado
  • PlatformIO — build system profissional
  • KiCad 7+ — design de PCB

Verificando a Instalação

Abra o terminal e confirme cada ferramenta:

# Python
python3 --version     # >= 3.10

# Git
git --version         # >= 2.30

# PlatformIO (via VSCode ou CLI)
pio --version

# KiCad
kicad --version       # 7.x ou 8.x

# VSCode
code --version

Todas devem retornar versões sem erro.

Git — Primeiros Passos

Configure seu nome e email (globais):

git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"

Clone o repositório do treinamento:

git clone https://github.com/organizacao/telemetry-trainee-program.git
cd telemetry-trainee-program

Criando sua Branch

Cada trainee trabalha na própria branch:

# Criar e mudar para a branch
git checkout -b seu-nome-sobrenome

# Verificar branches existentes
git branch

# Verificar status
git status

Nunca commitar na branch main ou master.

Commit — O Ciclo Básico

# Ver o que mudou
git status
git diff

# Adicionar arquivos ao stage
git add arquivo.py
git add .               # todos os arquivos

# Criar o commit
git commit -m "semana-00: adiciona script hello world"

# Enviar para o GitHub
git push -u origin seu-nome-sobrenome

Commits devem ser atômicos e com mensagens descritivas.

Pull Request "Hello World"

Objetivo: criar um arquivo hello.py na sua branch e abrir um PR.

# hello.py
def main():
    print("Hello from telemetry trainee!")

if __name__ == "__main__":
    main()
  1. Commit e push do arquivo
  2. Ir ao GitHub.com → abrir PR da sua branch para main
  3. Título: "Semana 00 — Nome Sobrenome"
  4. Adicionar revisores (mentores)

Estrutura do PR Ideal

Um bom PR contém:

  • Título claro e descritivo
  • Descrição do que foi feito e por quê
  • Prints/telas se aplicável
  • Checklist de verificação
## Descrição
Adiciona script inicial hello.py conforme requisito da Semana 00.

## Checklist
- [x] Código testado localmente
- [x] Segue as convenções do repositório
- [ ] Adicionados testes (quando aplicável)

Revisão de Código (Code Review)

Parte fundamental do processo:

  • Mentores revisam seu código e deixam comentários
  • Você responde e ajusta conforme necessário
  • Múltiplos ciclos de review até aprovação
  • Depois de aprovado: squash & merge

Não levar críticas como pessoais — revisão é sobre o código, não sobre você.

Fluxo do Treinamento

8 semanas, 8 módulos:

  • S00 — Onboarding & GitHub
  • S01 — Python básico (CSV, matplotlib)
  • S02 — Python aplicado (parsing, validação)
  • S03 — Arduino IDE (GPIO, I2C, Serial)
  • S04 — PlatformIO (estrutura profissional, testes)
  • S05 — Integração Arduino ↔ Python
  • S06 — KiCad básico (esquemático, PCB)
  • S07 — Flight Computer & Helike
  • S08 — Capstone (projeto final)

Dinâmica Semanal

  • Terça: Disponibilização do material
  • Quarta a Sexta: Estudo individual + exercícios
  • Sábado: Sessão síncrona (tirar dúvidas)
  • Domingo: Deadline para entrega
  • Segunda: Code review pelos mentores

Entregas são feitas via PR no repositório do treinamento.

Critérios de Avaliação

  • Completude dos exercícios propostos
  • Qualidade do código (organização, boas práticas)
  • Participação nas sessões e discussões
  • Evolução técnica entre as semanas
  • Colaboração com outros trainees

O objetivo não é "terminar", é aprender.
