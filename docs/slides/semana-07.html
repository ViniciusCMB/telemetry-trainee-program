O que é o Flight Computer?

O Flight Computer (FC) é o cérebro do foguete/satélite:

  • Processador principal (STM32, Teensy, RP2040)
  • Sensores: IMU, BMP280, GPS, magnetômetro
  • Atuadores: servo para paraquedas, buzzers, LEDs
  • Comunicação: LoRa (rádio), XBee, WiFi (solo)
  • Armazenamento: cartão SD para log de voo
  • Bateria: LiPo 2S-4S com BMS

FSM — Finite State Machine

O FC opera em uma máquina de estados:

        ┌──────────┐
        │  IDLE    │ (solo, esperando lancamento)
        └────┬─────┘
             │ comando de lancamento
        ┌────▼─────┐
        │  BOOST   │ (motor acionado)
        └────┬─────┘
             │ burnout
        ┌────▼─────┐
        │ COAST    │ (ascensao balistica)
        └────┬─────┘
             │ apogeu detectado
        ┌────▼─────┐
        │ DROGUE   │ (paraquedas piloto)
        └────┬─────┘
             │ altitude segura
        ┌────▼─────┐
        │ MAIN     │ (paraquedas principal)
        └────┬─────┘
             │ pouso detectado
        ┌────▼─────┐
        │  LANDED  │ (log final, beacon)
        └──────────┘

Telemetria no FC

O FC transmite pacotes de telemetria a cada ~50ms (20 Hz):

// Estrutura do pacote de telemetria
struct TelemetryPacket {
  uint32_t timestamp;    // ms desde o boot
  uint8_t  state;        // estado da FSM
  float    altitude;     // metros
  float    temperature;  // Celsius
  float    pressure;     // Pascal
  float    voltage;      // Volts (bateria)
  float    accel_x;      // m/s^2
  float    accel_y;
  float    accel_z;
  float    gyro_x;       // rad/s
  float    gyro_y;
  float    gyro_z;
  uint8_t  checksum;     // XOR de todos os bytes
} __attribute__((packed));

Helike PocketQube

Helike é um satélite no formato PocketQube (5×5×5 cm):

  • Padrão: 1P (5cm³) — ultra-compacto
  • Missão: coleta de dados ionosféricos
  • Órbita: LEO (Low Earth Orbit), ~400 km
  • Comunicação: LoRa ou AX.25 via rádio amador
  • CDB (Command & Data Base) — protocolo de bordo

CDB — Command & Data Base

O CDB define todos os parâmetros, comandos e formatos:

CDB define:
- IDs de cada pacote telemetrico
- Formatos de comando (TC)
- Formatos de telemetria (TM)
- Limites operacionais
- Tabela de calibracao dos sensores

Exemplo de entrada CDB:
TM_ID=0x01  PACOTE_ESTADO
  timestamp : uint32_t  (ms)
  state     : uint8_t   (FSM state)
  altitude  : float     (m)
  temp_bat  : float     (Celsius)
  voltage   : float     (V)

TC_ID=0xA0  COMANDO_RESET
  magic     : uint16_t  (0x5A5A)
  delay_ms  : uint32_t  (ms ate reset)

LoRa — Rádio Comunicação

LoRa (Long Range) é o link de rádio do Helike:

  • Frequência: 433 MHz ou 915 MHz (ISM band)
  • Alcance: até 10 km (solo) ou ~2000 km (satélite)
  • Taxa: 300 bps a 50 kbps
  • Modulação: Chirp Spread Spectrum (CSS)
  • Baixíssimo consumo de energia
// Configuracao LoRa tipica
#include <LoRa.h>

void setup() {
  LoRa.setPins(SS, RST, DIO0);
  LoRa.begin(433E6);     // 433 MHz
  LoRa.setSpreadingFactor(12);    // alcance maximo
  LoRa.setCodingRate4(8);         // robustez
  LoRa.setTxPower(20);            // 20 dBm
}

SRAB — Sistema de Recuperação

SRAB (Sistema de Recuperação e Aterrissagem do Bob):

  • Atuador: servo motor para liberação do paraquedas
  • Acionado pela FSM quando apogeu é detectado
  • Critérios: altitude decrescente por N amostras consecutivas
  • Redundância: timer de segurança (watchdog)
void check_apogee(float altitude) {
  static float alt_max = 0;
  static int amostras_descendo = 0;

  if (altitude > alt_max) {
    alt_max = altitude;
    amostras_descendo = 0;
  } else {
    amostras_descendo++;
  }

  // Apogeu confirmado apos 10 amostras descendo
  if (amostras_descendo >= 10 && estado == COAST) {
    estado = DROGUE;
    servo.write(180);  // libera paraquedas
    log_event("APOGEE_DETECTED");
  }
}

Oportunidades de Contribuição

Áreas onde você pode contribuir:

  • Firmware: drivers de sensores, FSM, controle
  • Python: pipeline de telemetria, dashboards em tempo real
  • Hardware: design de PCB, montagem, testes
  • LoRa: protocolo de comunicação, otimização de link
  • CDB: definição de parâmetros e comandos
  • Testes: validação de hardware, simulações
  • Documentação: manuais, wiki, relatórios de voo
  • SRAB: algoritmo de detecção de apogeu

Como Começar a Contribuir

  1. Escolha uma área de interesse
  2. Leia o código existente no repositório
  3. Pegue uma issue marcada como "good first issue"
  4. Discuta no Discord antes de implementar
  5. Abra um draft PR para feedback early
  6. Itere com os revisores

Não precisa saber tudo — a gente aprende junto.
