# Boas Praticas de PR

## Objetivo

Padronizar PRs para facilitar revisao e aprendizado.

## Checklist rapido

- Titulo claro e objetivo
- Descricao do que foi feito
- Como testar (passos simples)
- Evidencias (logs, prints, outputs)

## O que evitar

- PRs gigantescos sem foco
- Mudancas fora do escopo da issue
- Falta de README/instrucoes

## Dica de revisao

Prefira feedback pequeno e acionavel.
